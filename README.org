* Overview

Workshop materials for learning Veil, a Domain-Specific Language (DSL) for verified information-flow security in Dafny.

** What is Veil?

Veil is a DSL embedded in Dafny that enables developers to:

- Write security policies for information flow
- Verify that code adheres to these security policies
- Ensure confidentiality and integrity properties are maintained

* Quick Start

#+begin_src bash
# Complete setup
gmake setup

# Or step by step:
gmake dirs          # Create directories
gmake paper         # Download the Veil paper
gmake submodules    # Initialize git submodules
gmake check-lean    # Verify Lean 4 installation
#+end_src

* Prerequisites

** Lean 4

Veil verification requires Lean 4. Install via elan:

#+begin_src bash
curl https://raw.githubusercontent.com/leanprover/elan/master/elan-init.sh -sSf | sh
#+end_src

Verify installation:

#+begin_src bash
gmake check-lean
#+end_src

** Dafny

Veil is embedded in Dafny. See [[https://dafny.org/][Dafny documentation]] for installation.

* Make Targets

| Target            | Description                              |
|-------------------+------------------------------------------|
| =help=            | Show available targets                   |
| =setup=           | Complete setup (dirs, paper, lean check) |
| =paper=           | Download the Veil paper                  |
| =check-lean=      | Verify Lean 4 installation               |
| =check-tools=     | Check all required tools                 |
| =submodules=      | Initialize/update git submodules         |
| =submodules-check= | Verify submodule integrity               |
| =status=          | Show current setup status                |
| =tutorial-init=   | Create tutorial directory structure      |
